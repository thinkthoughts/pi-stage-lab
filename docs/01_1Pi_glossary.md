# 01 — 1Pi Glossary: Identity Extend

## Core rule

A valid identity remains consistent through interaction.

## Identity triplet

- 0Pi: define measurable identity
- 1Pi: test identity through interaction
- 2Pi: preserve identity across constraint

## Template contract

Every Pi-stage notebook should keep the same output pattern:

- docs/{stage}_glossary.md
- docs/{stage}_notes.md
- docs/{stage}_readme_snippet.md

## Generic terms

| TPL / CGCS term | Generic meaning | Later translations |
|---|---|---|
| identity | measurable structure | photon, molecule, field, state, cycle variable |
| extend | test identity through interaction | measurement, exchange, perturbation |
| valid construction / VC | identity persists through interaction | physics/chemistry model |
| general-origin story / GOS | identity preserved through constraints | system-aware framing |
| invalid assignment / IA | interaction collapses identity into interpretation | slogan collapse |
| zero-origin story / ZOS | claim pretending interaction can redefine identity without measurement | public-language shortcut |
| constraint signal | measurable variable connecting system states | angle, flux, concentration, energy, rate |
| CGCS | constraint-gated comprehension score | claim-quality score |

## Translation boundary

1Pi is grammar, not application. Do not lock 1Pi to one domain.

Translate into photons, CO2, O2, climate, chemistry, or public claims later in bridge docs and later notebooks.
